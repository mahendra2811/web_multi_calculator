# Calculator Priorities — SEO Investment Tiers

> **Purpose**: Decide where to spend the limited "write 1,200 words of original SEO content + add Block-4 visualization" budget. Calculator.net's moat is content depth, not calculator count. We have 387 calculators; we need to upgrade the **top ~100** (P-0 + P-1 + P-2) to the full 6-block standard from `CLAUDE.md`. The rest stay functional but lean.
>
> **Last reviewed**: May 2026. Update quarterly with real Search Console traffic data.

## Methodology

Each calculator is graded P-0 → P-3 based on a weighted blend of three signals:

| Signal                                 | Weight | Source                                                                                                                                         |
| -------------------------------------- | ------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Estimated global monthly search volume | 50%    | SEMrush / Google Keyword Planner public data for the calculator's primary keyword (e.g., "EMI calculator", "SIP calculator", "BMI calculator") |
| India-market relevance                 | 30%    | Whether the topic is India-specific or has strong Indian search intent (PPF, EPF, GST, HRA win here; "mortgage UK" loses)                      |
| Conversion / engagement potential      | 20%    | Does the calc keep users on the page (amortization, charts, multi-input forms)? Does it lead to other calcs?                                   |

**Tier definition:**

- **P-0** — Top 20. The flagship calculators. Each gets the **full 6-block treatment**: 1,200-1,500 word MDX content, visualization, curated FAQs (8-12), hand-picked related calcs, share-friendly URLs.
- **P-1** — Next 30. Each gets **800-1,000 word MDX content**, visualization where applicable, 6-8 curated FAQs.
- **P-2** — Next 50. Each gets **500-800 word MDX content** (focused: formula + example + when-to-use), 5 curated FAQs.
- **P-3** — Remaining 287. **No MDX**, auto-generated FAQs are enough. Lean schema-driven page. We let traffic data tell us which ones graduate to P-2 later.

**Total content writing budget**:

- P-0: 20 × 1,300 words ≈ **26,000 words**
- P-1: 30 × 900 words ≈ **27,000 words**
- P-2: 50 × 650 words ≈ **32,500 words**
- **Grand total: ~85,000 words of original SEO content.** This is a multi-sprint effort, not a one-shot.

## Content writing guide (apply to P-0/P-1/P-2)

When writing the MDX content for a calculator, follow this exact section order. Calculator.net does this on every page and it's why they rank.

### Required sections (in order)

```mdx
# {Calculator Name}

> _The H1 is auto-rendered by `CalculatorShell` — DON'T duplicate it in the MDX._

## What is {X}?

2–3 plain-English paragraphs. Define the concept, the use case, who needs it.
End with one provocative or counterintuitive sentence to hook the reader.

## How is {X} calculated?

Show the formula in `code` blocks. Define every variable.
For finance/health: include both SI and (where relevant) US-customary forms.

## Worked example

Take real numbers (Indian context first). Walk through the calculation step by step.
Use a small markdown table for the intermediate steps. End with the final answer.

## Components and inputs explained

H3 for each major input. Explain what it means, where to get the number,
common pitfalls. Use bullets for the gotchas.

## Common variants and types

Country variants (Indian PPF vs UK ISA), method variants (Epley vs Brzycki 1RM),
slab variants (M15 vs M20 vs M25 concrete). Use tables aggressively.

## Considerations

What affects the result? Edge cases. Inflation impact (for long-horizon finance).
Tax treatment (for India finance). Real-world calibration vs ideal-case math.

## Limitations

Honest caveats. BMI doesn't measure muscle. Free fall ignores air resistance.
This section builds trust. Don't skip it.

## Related calculators

Mention 6 by name with a one-line connection.
The actual link cards are rendered automatically by <RelatedCalculators>.

## Final note (1 paragraph)

Reinforce when to use this calc, when to ditch it for a CA / doctor / pro.
```

### Length targets

| Tier | Word count  | Sections required                                                                        |
| ---- | ----------- | ---------------------------------------------------------------------------------------- |
| P-0  | 1,200–1,500 | All 9 above + extra Variants table                                                       |
| P-1  | 800–1,000   | All 9, but Variants can be brief                                                         |
| P-2  | 500–800     | Skip "Common variants" if not applicable. Worked example + Considerations are mandatory. |

### Style guide (very important)

1. **Write like a knowledgeable friend, not a Wikipedia entry.** Use first/second person ("you'll notice", "I'd recommend"). Calculator.net is dry; we win on tone.
2. **Lead with the answer.** Don't make the reader scroll for the formula. Section 2 (How is X calculated) should be done by word 200.
3. **Indian context first** for finance calcs (₹ amounts, Indian tax regime, PPF slabs). Add US/UK/global variants in the Variants section.
4. **Use tables for comparisons.** Markdown tables render beautifully in our prose styles. Comparisons (regime A vs B, formula A vs B) belong in tables.
5. **No filler.** If a section adds nothing, skip it. We have a word target, not a minimum.
6. **Link liberally to other calculators.** Inline `[Mortgage Calculator](/calculator/mortgage)`. Google sees these as topical clustering.
7. **End each H2 section with one takeaway sentence** in bold or italics. Helps the reader skim.

## When to re-prioritize

- **Quarterly** — pull Search Console "Top Queries" + "Top Pages" reports. Promote calcs with >5K impressions to P-2; demote any P-2 with <500 impressions after 6 months of content.
- **After major news events** — Budget Day shifts tax-calc demand; crypto bull runs spike crypto calcs; rate-cut cycles spike refinance / mortgage calcs.
- **After a viral blog** — if a CalcMaster blog post drives traffic to a P-3 calc, promote it.
